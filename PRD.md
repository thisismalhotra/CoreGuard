# Product Requirements Document (PRD): Core-Guard MVP **Version:** 1.0 **Status:** MVP (Glass Box Simulation) **Target Vertical:** Hardware Electronics Manufacturing (Project FL-001)  ## 1. Executive Summary Core-Guard is an autonomous supply chain operating system. It replaces manual procurement workflows with a network of specialized AI Agents.  **The MVP Goal:** Build a "Glass Box" simulation where a user can trigger a supply chain crisis (e.g., a demand spike) and watch AI agents autonomously detect, analyze, negotiate, and execute a solution (Purchase Order) without human intervention.  ## 2. User Personas * **The Operator (User):** A COO or Supply Chain Manager who wants to visualize the "Nervous System" of their factory. They interact with the Command Center Dashboard. * **The Agents (System):** Autonomous entities that perform specific roles (Demand Sensing, MRP Calculation, Purchasing).  ## 3. Functional Requirements (The Scope)  ### 3.1 The Dashboard (Command Center) * **Visuals:** A futuristic, dark-mode dashboard (Next.js/Tailwind). * **Tabs:** 1.  **Network Status:** High-level KPIs (Inventory Health, Active Threads, Automation Rate).     2.  **Live Logs:** A scrolling terminal showing Agent-to-Agent communication in real-time.     3.  **Digital Dock:** Visual representation of incoming quality inspections.     4.  **God Mode (Simulation):** A control panel with buttons to inject chaos (e.g., "Simulate 300% Spike").  ### 3.2 The Simulation Engine ("God Mode") The user must be able to trigger specific scenarios via API/UI: * **Scenario A (Demand Spike):** Simulate a 300% increase in sales for the "Tactical Flashlight" (FL-001-T). * **Scenario B (Supply Shock):** Simulate a fire at a primary supplier (AluForge). * **Scenario C (Quality Fail):** Simulate a batch of Chassis failing inspection at the dock.  ### 3.3 The Agent Squad (Logic Requirements) The system must simulate the following agents interacting:  1.  **Aura (Demand Agent):**     * *Trigger:* Monitors sales data (mock CSV/API).     * *Action:* Detects when `Sales > Forecast * 1.2`. Fires `DEMAND_SPIKE` event. 2.  **Core-Guard (MRP Agent):**     * *Trigger:* Receives `DEMAND_SPIKE`.     * *Logic:* Calculates `Net_Requirement`. Checks Inventory.     * *Action:* If shortage exists, checks if inventory can be reallocated from lower-priority variants. If not, triggers Ghost-Writer. 3.  **Ghost-Writer (Procurement Agent):**     * *Trigger:* Receives `BUY_ORDER` from Core-Guard.     * *Logic:* Checks cost against the "Financial Constitution" (Max Spend $5,000).     * *Action:* Generates a PDF Purchase Order. Logs the transaction. 4.  **Eagle-Eye (Quality Agent):**     * *Trigger:* Simulates receiving a physical shipment.     * *Logic:* Compares input data against "CAD Specs" (Vector DB).     * *Action:* Passes or Fails the batch.  ## 4. Data Requirements (The Ground Truth) The system must be seeded with the **FL-001 Flashlight** dataset: * **BOM:** 5 Variants (Standard, Tactical, etc.) sharing common parts. * **Inventory:** Real-time counts for CH-101, SW-303, LNS-505. * **Suppliers:** Mock database of 22 vendors with lead times and costs.  ## 5. Non-Functional Requirements * **Latency:** Agent reactions must appear on the dashboard within 2 seconds. * **Persistence:** Transactions must be saved to a SQL database. * **Visibility:** Every decision made by an AI must be logged in a human-readable format.
